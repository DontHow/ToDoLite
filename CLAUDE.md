# CLAUDE.md

This file gives coding guidance for agents working in this repository.

## Project

**办他 / TodoLite** is a Local First SwiftUI task app for iOS and macOS, with an iOS Widget and a macOS menu bar extra. The app stores user data as JSON files in iCloud Documents when available, falling back to the local Documents directory. SQLite is used only for FTS search indexing.

The current app is no longer the older broad MVP spec. Match the code, not old product notes.

## Build

The Xcode project is generated by XcodeGen from `project.yml`.

```bash
xcodegen generate
```

Build commands:

```bash
xcodebuild -project TodoLite.xcodeproj \
  -scheme TodoLite-iOS \
  -destination 'platform=iOS Simulator,name=iPhone 16' \
  build

xcodebuild -project TodoLite.xcodeproj \
  -scheme TodoLite-macOS \
  -destination 'platform=macOS' \
  build
```

Tests:

```bash
xcodebuild -project TodoLite.xcodeproj \
  -scheme TodoLite-iOS \
  -destination 'platform=iOS Simulator,name=iPhone 16' \
  test
```

Targets from `project.yml`:

- `TodoLite-iOS`: iOS app, deployment target 17.0
- `TodoLite-macOS`: macOS app, deployment target 14.0
- `TodoLiteWidget`: iOS WidgetKit extension
- `TodoLiteTests`: iOS unit tests

Do not hand-edit `TodoLite.xcodeproj/project.pbxproj` when the change belongs in `project.yml`.

## Architecture

Data flow:

```text
View -> TodoStore (@Observable) -> Repository actor -> FileSystemManager actor -> JSON files
```

Views should call `TodoStore` APIs. Do not make feature views read or write JSON files directly.

Important components:

- `TodoLite/App/TodoLiteApp.swift`: app entry. Creates `ContentView`, sets up directories, loads all data, starts iCloud monitoring, and adds the macOS `MenuBarExtra`.
- `TodoLite/App/ContentView.swift`: platform navigation. iOS uses `TabView`; macOS uses `NavigationSplitView` plus keyboard shortcuts.
- `TodoLite/Shared/Store/TodoStore.swift`: observable app state and public async CRUD surface.
- `TodoLite/Shared/Repository/*.swift`: actor repositories for tasks, projects, tags, focus set, and LLM config.
- `TodoLite/Shared/FileSystem/FileSystemManager.swift`: JSON read/write/list/delete through `NSFileCoordinator`; creates app directories; falls back from iCloud to local Documents.
- `TodoLite/Shared/FileSystem/iCloudSyncManager.swift`: `NSMetadataQuery` monitor for downloaded JSON changes.
- `TodoLite/Shared/Search/SearchIndexer.swift`: SQLite FTS5 contentless index in Application Support.
- `TodoLite/Shared/Parser/TodoParser.swift`: rule parser for quick entry.
- `TodoLite/Shared/LLM/*.swift`: OpenAI-compatible chat service and task parser.
- `TodoLite/Shared/Widget/WidgetDataStore.swift`: writes today-focus summaries to App Group defaults for the widget.

## Current Product Shape

Primary screens:

- Today: Focus, Suggested, Overdue, Upcoming sections
- Inbox: tasks with `status == .inbox`
- Board: three columns, `inbox`, `doing`, `done`
- Search: SQLite FTS results over title, description, project, tags
- Projects: project CRUD
- Tags: tag management
- Done: completed tasks grouped by week, with optional LLM weekly report generation
- Archive: archived tasks
- Settings: font size and LLM config

iOS top-level navigation contains Today, Inbox, Board, Search, Settings. macOS sidebar also exposes Projects, Tags, Done, and Archive.

## Models

`TodoItem` fields:

- `id`
- `title`
- `description`
- `status`
- `projectId`
- `tagIds`
- `scheduledAt`
- `dueAt`
- `createdAt`
- `updatedAt`
- `completedAt`
- `version`

Current statuses in `TodoStatus`:

- `inbox`
- `doing`
- `done`
- `archived`

Legacy status decoding is intentionally supported:

- `next` -> `doing`
- `waiting` / `blocked` / `someday` -> `doing`
- `cancelled` -> `archived`
- unknown -> `inbox`

Do not reintroduce old statuses unless the product and migrations are updated deliberately.

Defaults:

- New task: `status = .inbox`, `version = 1`
- New project: emoji `📁`, color `#007AFF`, `isArchived = false`
- New tag: color `#FF9500`
- LLM config: base URL `https://api.openai.com/v1`, model `gpt-4o-mini`

## Today Semantics

Today Focus is not stored on `TodoItem`. It is stored as `FocusSet` in `meta/focus_yyyy-mm-dd.json`.

Derived collections in `TodoStore`:

- `focusTodos`: IDs in the current `FocusSet`, excluding done and archived
- `suggestedTodos`: due today, not focused, excluding done and archived
- `overdueTodos`: due before today, not focused, excluding done and archived
- `upcomingTodos`: due tomorrow or later, not focused, excluding done and archived

Keep these concepts separate:

- `scheduledAt`: planned date field on a task
- `dueAt`: deadline field used by current Today suggestions
- `FocusSet`: explicit today attention list

The current UI mostly uses `dueAt`; do not claim scheduled-date behavior unless implemented.

## Files On Disk

Expected app data layout:

```text
TodoLite/
  tasks/task_<id>.json
  projects/project_<id>.json
  tags/tag_<id>.json
  trash/
  archive/
  conflicts/
  meta/focus_yyyy-mm-dd.json
  config/llm_config.json
```

`FileSystemManager` creates these directories. `SearchIndexer` writes `search_index.sqlite` under Application Support, not into the source-of-truth directory.

## Persistence Rules

- JSON files are the source of truth.
- Search index is disposable and rebuilt from current store data.
- `TodoRepository.save` checks versions and writes conflict backups before throwing `FileSystemError.conflictDetected`.
- Saves increment `version` and refresh `updatedAt`.
- Task deletion currently removes the JSON file; archive changes `status` to `.archived`.
- LLM never writes files directly. It returns drafts/text; persistence still goes through store/repository.

## Quick Entry And LLM

Rule parser syntax:

- `@project`
- `#tag`
- `^date`

Dates are resolved by `DateResolver` and include `today`, `tomorrow`, `weekend`, `next week`, weekdays, and simple numeric date formats.

Creating a task from quick entry can create missing projects/tags by name. The default new-task due date in `CreateTodoView` is three weekdays from today when no parsed due date is supplied.

LLM parsing uses `LLMParser` -> `LLMService.chat` against `baseURL + /chat/completions`.

## Widget And Menu Bar

`WidgetDataStore.sync` writes focus-task count and the first three focus task titles to `UserDefaults(suiteName: "group.com.donghao.TodoLite")`, then reloads the widget timeline when WidgetKit is available.

The macOS menu bar extra shows up to five focus task titles, a new-task command, open-app command, and quit.

## UI Guidelines

- Use SwiftUI and SF Symbols.
- Keep iOS and macOS platform-specific APIs behind `#if os(iOS)` / `#if os(macOS)`.
- Prefer existing shared components: `TaskListView`, `TodoListCard`, `TagChip`, `FlowLayout`, `OptionRow`, `CardButtonStyle`, `EmptyStateView`.
- Use the existing color helpers in `Color+Card.swift` and `Color+Hex.swift`.
- Preserve the app's native, quiet productivity feel. Avoid marketing-style screens, heavy decorative effects, or cartoon visuals.
- Use app-specific Chinese copy for user-facing text unless there is a clear reason not to.

## Agent Workflow

- Inspect current code before changing docs or behavior.
- Respect uncommitted user changes. Do not revert unrelated files.
- Do not auto-commit unless the user asks.
- After code changes, run the most relevant build or test command before claiming success. For documentation-only edits, at minimum inspect the changed files and `git diff --check`.
- If changing `project.yml`, run `xcodegen generate` and then verify the generated project change.
